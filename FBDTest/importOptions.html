Animation XML options

  • solnType (attribute)
    Identifies an object as a solution area. solnType must be paired with a solnObj
    Options:
    AllInside = solution is satisfied when all of an answer object is inside the solution area. Direction is not considered
    AllInsideDir = solution is satisfied when all of an answer object is inside the solution area and pointed in the same direction as the solution object
    AllInsideFill = solution is satisfied when all of an answer object is inside the solution area and mostly fills the solution. Currently only works with a curve object type. If used with other objects the type acts like AllInside.
    AnyInside = solution is satisfied when any of the answer object is inside solution object
    NoneInside = solution is satisfied if no answer objects matching the solnObj type are inside the solution object's area
    *Unique = the answer object satisfying this criterion can only be used once. Allows to have to unique solutions objects on top of each other.
  • solnObj (attribute)
    Identifies the type of answer object measured by a solution object object in a solution area. solnObj must be paired with a solnType
    Options:
    vector = only vector answer objects are checked against the solution object
    curve = only curve answer objects are checked against the solution object
    userdefined = only answer objects matching the userdefined names are checked against the solnObj
  • innerHTML
    When an object is a solution, the description for that object is in the object's innerHTML
  • graph (objType)
    A custom animator objType used for scaling. Object must be an animator "box"
    graphScale (attribute)
    graphScale="[xmin, ymin, xmax, ymax]" specifies the engineering units corresponding to the rectangle's location and size
    graphCursor (attribute)

    Options:
    "true"= the corresponding engineering units are displayed when the mouse is moved inside the object's rectange
    "false"= the corresponding engineering units are not displayed when the mouse is moved inside the object's rectange. Option used for scaling from pixels to engineering units when the graph object is part of a group.
  • polyline (objType)
    A custom animator objType used for drawing polygons (open or closed).
    polyPoints (attribute)
    polyPoints="x1,y1, x2,y2, ..." where x#,y# are the nodes of the polygon.
    closed (attribute)

    Options:
    "true"= the polygon is closed, automatically connecting the first and last points in polyPoint
    "false"= the polygon is open (a line)
  • group (objType)
    A group of objects with the same syntax and meaning as in the Animator. However groups of objects can be moved or rotated by changing the groups parameters
    If a graph is part of a group, the scaling for the graph is applied to all parameter changes for items inside the group.
  • snap (objType)
    A custom animator type based on a box. The snap identifies labels on a figure used for positioning objects like vectors. The size of the snap's box determines the snaps tolerance. The objName is the value shown in dropdowns using the snap values. The layer position in Animator determines the snap order in dropdowns.

Notes:

  1. All objects must have a unique objNum and objName
  2. Object parameter can be changed using ObjectName_attribute=value; Ex. box1_x=10; changes the x value of object named box1 to 10.
    Images are position relative to their upper left coordinates. Consider centering images.
  3. Groups with a graph object use the graph's scaling when changing object parameters
