Notes:

  1. The layouts below are for Nise and Groover. The additional Moran and LSaS layouts should be verified by their assigned authors.
  2. Do we want the keyboard layouts to have the same key layout pattern for all books, or for all layouts within a book?
  3. Use \bar or \overline? I like the look of \overline better but \bar better matches the desired semantics.
  4. Should we allow the default shift operators, Ex: '[1]' or use '1'. The keyboards below use the default [*] for some key, like number keys, so have additional actions if you hold down the shift or long press the key.
  5. The create matrix button, M, in Nise should open a dialog that lets you create a matrix of arbitrary size - up to a reasonable size. Options are:
    1. Open the default mathfield matrix menu.
    2. Use keyboard variants so the user press and holds the key for options.
    3. Create a separate Matrix layout with keys for common matrix sizes.
    4. Use the "Arrays" commands to let users build arrays in the edit field.
